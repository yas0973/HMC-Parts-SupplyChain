## 1. 프로젝트 개요

| 항목 | 내용 |
| :--- | :--- |
| 프로젝트명 | HMC-SCM (부품 공급망관리·조달 시스템) |
| 목적 | 협력사 등록부터 입찰·납품·평가·정산까지 조달 전 과정을 단일 웹으로 통합 관리 |
| 기대효과 | 수기/분산 관리 제거, 입찰 투명성 확보, 협력사 평가 데이터화, 납품 효율화 |

## 2. 비즈니스 배경 (BRD 요약)

- 부품 조달 업무가 문서·메일·전화로 분산되어 추적성과 투명성이 낮음
- 협력사 성과(납기·품질) 데이터가 축적되지 않아 객관적 선정 근거 부족
- 생산 계획·납품 동선 최적화가 수작업 → 비용·시간 낭비

## 3. 프로젝트 범위 (System Scope)

### 3.1 포함 범위 (In-Scope)
- 회원/협력사 가입·승인·권한 관리
- 공지사항 게시/열람
- 입찰 등록·공고·응찰·낙찰
- 품목/부품 마스터 관리 및 검색
- 납품 등록·현황 관리
- 협력사 평가 및 리포트
- 정산
- 생산 최적화(유전알고리즘), 납품 최적경로(TSP)

### 3.2 제외 범위 (Out-of-Scope)
- 회계/ERP 전표 연동 *(확인 필요)*
- 모바일 앱 *(현재 웹 only)*
- 전자결제/PG 연동 *(확인 필요)*

## 6. 제약 및 가정 (Constraints & Assumptions)

- 런타임: Apache Tomcat 9.0 (Servlet/JSP)
- DBMS: MySQL, 스키마 `HMC_SCM`
- 외부 의존: Naver Maps API (경로 시각화)
- 가정: 사내망 또는 제한된 사용자 대상 운영 *(확인 필요)*
